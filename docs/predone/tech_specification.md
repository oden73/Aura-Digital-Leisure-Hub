# Интеллектуальная система интеграции развлекательных сервисов (на основе предпочтений пользователя)

## ОБЗОР

Проект направлен на решение проблемы «усталости от выбора» в условиях фрагментации цифрового контента. Сегодня пользователь вынужден переключаться между десятками сервисов (Steam, Кинопоиск, LiveLib и др.), чтобы решить, как провести досуг. Система выступает единым интеллектуальным хабом, который агрегирует данные об интересах пользователя и предлагает персонализированный контент (игры, книги, кино) в одном интерфейсе, используя нейросетевые модели для глубокого анализа предпочтений и кросс-медийных связей. 

## ЦЕЛИ

1. Централизация досуга. Создать единую точку доступа к рекомендациям из разных сфер развлечений.
2. Кросс-платформенная персонализация. Реализовать алгоритм, предлагающий контент в одной сфере (например, книги) на основе интересов в другой (например, игры).
3. Снижение когнитивной нагрузки. Обеспечить быстрый подбор развлечений под текущее настроение пользователя.
4. Интерактивный поиск. Интегрировать интеллектуального ИИ-ассистента для обработки сложных и нестандартных запросов пользователя.

## ОСНОВНЫЕ ФУНКЦИОНАЛЬНЫЕ МОДУЛИ

1. Модуль агрегации данных. Система синхронизации с внешними платформами (Steam, профили кино- и книжных сервисов) для формирования единого профиля интересов.
2. Рекомендательный движок. Три независимые ленты (Кино/Книги/Игры) с поддержкой функции «Mood Selector» (подстройка под текущее состояние).
3. AI-консультант. Чат-интерфейс на базе LLM (Large Language Model), способный аргументированно предлагать контент и отвечать на узкоспециализированные запросы.
4. Аналитический хаб. Экран статистики, визуализирующий распределение интересов пользователя и динамику его предпочтений.

## ТЕХНИЧЕСКИЕ ПОДРОБНОСТИ

- Backend (Core) - Go (Golang). Ответственен за основную бизнес-логику, агрегацию данных из внешних API (Steam, IMDb и др.), управление профилями пользователей и высокую скорость обработки запросов.
- Backend (AI Engine) - Python. Специализированный сервис для реализации интеллектуальных функций: генерация персональных подборок, работа с векторными представлениями контента и интеграция с языковыми моделями (LLM).
- Frontend - React. Обеспечивает интерактивный интерфейс пользователя и динамическое отображение лент рекомендаций.
- Межсервисное взаимодействие - использование gRPC или REST протоколов для передачи данных между Go и Python модулями.
- Хранение данных - PostgreSQL для структурированной информации и ChromaDB для векторного поиска сходства контента.
